# Lead Playbook — Using AGENTS.md + Session Templates (v2)

**Last Updated:** 2026-02-28

## Quick start
1) Create `SESSION-{ID}-KICKOFF` and fill it in.
2) Pick mode + comms policy for this session.
3) Staff only the roles you need (PM/BA optional).
4) Issue Session Contracts to each teammate with file scopes.
5) Require the standard deliverable format.
6) Close with `{YYMMDD}_SESSION-{ID}-REPORT.md` and record verification.

## When to require plan approval
Use plan approval for:
- DB/schema changes
- Auth/security changes
- Multi-module interface changes
- Broad refactors

## Conflict avoidance
- Assign explicit file ownership per teammate.
- Use HYBRID comms as default; allow direct messaging only for dependencies/conflicts/review.
